## Critical architecture

- **Three-layer**: `Routes (modules/) → Services → Repositories`. Repositories extend `AuditableRepository` which sets `app.user_id` for RLS.
- **RLS is the sole access control mechanism** — never disable, never bypass. 174+ policies across all tables.
- **PostgreSQL 18.3 Windows bug**: `FOR INSERT ... WITH CHECK` policies fail silently. Workaround: use `SECURITY DEFINER` functions (see `backend/seed/33-fix-register-rls.sql`) or `FOR ALL USING (true) WITH CHECK (...)`.
- **Context propagation**: `AsyncLocalStorage` in `middleware/context.ts` sets `app.user_id` per-request. The Pool connect handler reads it and calls `SET SESSION app.user_id = $1`. This is mandatory for RLS to work.
- **Frontend SDK** (`sdk/domains/`) is generated from OpenAPI contract via Orval. Do not hand-edit SDK files — edit the OpenAPI spec and regenerate.

## Database setup

```bash
# Drop/create DB + schemas
psql -U postgres -d ethics_db -f "DDL Script.sql"

# Tables
psql -U postgres -d ethics_db -f ethics_db_tables.sql

# Functions
psql -U postgres -d ethics_db -f ethics_db_functions.sql

# Constraints
psql -U postgres -d ethics_db -f ethics_db_tables_constraints.sql
```

Seeds in `backend/seed/` are applied in numeric order (00 through 33+) and are **not** run automatically. Must apply manually:
```bash
for f in backend/seed/*.sql; do psql -U postgres -d ethics_db -f "$f"; done
```

## Testing quirks

- **Integration tests** in `backend/src/test/` hardcode port **3000**, but dev server runs on **8080**. Pre-existing mismatch; update test env or skip when running locally.
- **E2E tests** (`scripts/e2e-full-test.ps1`) target `http://localhost:8080/api/v1` and require running backend + seeded database.
- **Frontend tests** use `jsdom` — no real browser. E2E tests use Playwright.
- CI runs tests in order: typecheck → build → test → e2e (depends on backend + frontend passing).

## Security constraints

- **Never disable RLS** (`ALTER TABLE ... DISABLE ROW LEVEL SECURITY`). All fixes must work within RLS.
- The audit trigger `system.fn_log_audit()` expects `app.user_id` to be set. Always called within a database session that has set this parameter.
- `Soft delete` columns (`deleted_at`, `deleted_by`) are on ~80 transaction tables. RLS policies may need `is_active_row()` filtering.

## RLS fixes applied

### 1. Registration RLS (`33-fix-register-rls.sql`)
- **Problem**: PostgreSQL 18.3 Windows bug: `FOR INSERT WITH CHECK` policies fail silently for `security.users`.
- **Fix**: `SECURITY DEFINER` function `security.fn_register_user(...)` bypasses RLS for registration only.
- **Status**: Applied, verified.

### 2. Documents INSERT RLS (`34-documents-insert-rls.sql`)
- **Problem**: No INSERT policy on `documents.documents` with RLS enabled → all document uploads denied.
- **Fix**: Policy `documents_insert_policy` on `documents.documents` with `FOR INSERT WITH CHECK`:
  - Admin (`fn_is_admin`): unrestricted.
  - Regular user: `uploaded_by = app.user_id` AND entity ownership check for `Application` type.
- **Testing evidence**: Confirmed `FOR INSERT WITH CHECK` works on PostgreSQL 18.3 Windows (no silent failure for this table).
- **Status**: Applied, verified (3/3 tests pass).
