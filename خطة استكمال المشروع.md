## المرحلة 0: ترتيب قاعدة البيانات (Database Housekeeping)

### الوضع الحالي
- جميع الـ Schemas الـ 14 مملوكة للمستخدم `postgres` وليس `ethics_owner`
- المستخدم `ethics_owner` ليس لديه صلاحيات USAGE على أي Schema
- بعض الجداول تفتقد Foreign Keys (حسب ملاحظات DDL)

### المطلوب

#### 0.1 تغيير ملكية الـ Schemas
```sql
ALTER SCHEMA security OWNER TO ethics_owner;
ALTER SCHEMA core OWNER TO ethics_owner;
ALTER SCHEMA workflow OWNER TO ethics_owner;
ALTER SCHEMA committee OWNER TO ethics_owner;
ALTER SCHEMA documents OWNER TO ethics_owner;
ALTER SCHEMA monitoring OWNER TO ethics_owner;
ALTER SCHEMA safety OWNER TO ethics_owner;
ALTER SCHEMA communication OWNER TO ethics_owner;
ALTER SCHEMA audit OWNER TO ethics_owner;
ALTER SCHEMA reporting OWNER TO ethics_owner;
ALTER SCHEMA reference OWNER TO ethics_owner;
ALTER SCHEMA integration OWNER TO ethics_owner;
ALTER SCHEMA system OWNER TO ethics_owner;
```

#### 0.2 منح الصلاحيات الأساسية
- GRANT USAGE, CREATE ON SCHEMA ... TO ethics_migration
- GRANT USAGE ON SCHEMA ... TO ethics_app, ethics_workflow, إلخ
- GRANT SELECT, INSERT, UPDATE, DELETE على الجداول حسب الدور

#### 0.3 إضافة Foreign Keys المفقودة
- ربط `committee.committees.committee_type_id` ← `committee.committee_types(id)` (ملاحظة موجودة في DDL)
- إضافة `committee_member_roles` (جدول ربط بين committee_members و committee_roles)

#### 0.4 ضبط Default Privileges
```sql
ALTER DEFAULT PRIVILEGES IN SCHEMA core GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO ethics_app;
ALTER DEFAULT PRIVILEGES IN SCHEMA workflow GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO ethics_app;
-- إلخ لكل Schema
```

## المرحلة 4: Row-Level Security (RLS)

### الوضع الحالي
- 0 جدول مفعل عليه RLS
- 0 Policy

### المطلوب

#### 4.1 تفعيل RLS على الجداول الحساسة
| الجدول | سياسة RLS |
|---|---|
| core.applications | الباحث يرى فقط طلباته |
| core.projects | الباحث يرى فقط مشاريعه |
| committee.reviews | المراجع يرى فقط المراجعات المخصصة له |
| committee.meetings | أعضاء اللجنة فقط |
| security.users | المستخدم يرى فقط بياناته |

#### 4.2 إنشاء Policies
```sql
ALTER TABLE core.applications ENABLE ROW LEVEL SECURITY;
CREATE POLICY applications_policy ON core.applications
    FOR SELECT
    USING (submitted_by = current_setting('app.user_id')::BIGINT
           OR EXISTS (SELECT 1 FROM security.user_roles WHERE user_id = current_setting('app.user_id')::BIGINT AND role_id IN (SELECT id FROM security.roles WHERE code IN ('SUPER_ADMIN', 'ETHICS_ADMIN'))));
```
