## أولوية متوسطة

### 3. shadcn/ui (مكتبة مكونات UI)
- **الوصف**: إضافة shadcn/ui للمكونات الجاهزة (جداول، نماذج، أزرار، بطاقات)
- **المكتبات**: `shadcn/ui` + `class-variance-authority` + `clsx` + `tailwind-merge` + `tailwindcss-animate`
- **الموقع**: `frontend/src/components/ui/`
- **الجهد**: ~3 ساعات
- **ملاحظات**: يتطلب إعداد `components.json` + تكوين Tailwind للإضافات

### 4. React Hook Form كامل لجميع النماذج
- **الوصف**: توحيد جميع نماذج الإدخال باستخدام React Hook Form مع Zod validation
- **المكتبات**: `react-hook-form` + `@hookform/resolvers` (موجودان مسبقاً)
- **الجهد**: ~ساعتان لكل صفحة

## أولوية منخفضة (تتطلب بنية تحتية)

### 5. MinIO + Multer (تخزين الملفات)
- **الوصف**: استبدال التخزين المحلي للملفات بـ MinIO (متوافق مع S3)
- **المتطلبات**: خادم MinIO منفصل
- **المكتبات**: `@aws-sdk/client-s3`
- **الجهد**: ~3 ساعات
- **ملاحظات**: يحتاج إعداد MinIO server + تكوين buckets + سياسات الوصول

### 6. Redis + BullMQ (تخزين مؤقت + مهام خلفية)
- **الوصف**: إضافة Redis للتخزين المؤقت و BullMQ للمهام المجدولة (إشعارات، تقارير)
- **المتطلبات**: خادم Redis منفصل
- **المكتبات**: `ioredis`, `bullmq`
- **الجهد**: ~4 ساعات
- **ملاحظات**: يمكن البدء بـ Redis وحده للتخزين المؤقت، ثم إضافة BullMQ لاحقاً

### 7. Prometheus + Grafana (مراقبة)
- **الوصف**: تصدير metrics من التطبيق + لوحة مراقبة Grafana
- **المتطلبات**: Prometheus server + Grafana
- **المكتبات**: `prom-client`
- **الجهد**: ~4 ساعات
- **ملاحظات**: مفيد بعد deployment في الإنتاج

### 8. Drizzle ORM (اختياري)
- **الوصف**: إضافة طبقة ORM اختيارية مع SQL المباشر
- **التقييم**: قيمة محدودة لأن SQL المباشر عبر pg مقيم 10/10
- **الجهد**: ~6 ساعات
- **ملاحظات**: قد يكون مفيداً للاستعلامات البسيطة دون كتابة SQL يدوي
